Lession 1 - Units

Alternatives to px

As developer's we have a tendency to copy a pixel value from a design and use that in the site, only to find that it does not work when we switch to a different screen size. This lession explores what options we have as developers to use different css units types.

Below we have a box with options available on a width parameter. In the past, percentages were used to create even columns (50% for 2, 33.3% for 3, etc.) we will see that now we have better options for this.

Lession 3 - Grids

Grids

There are a number of options for a grid column or row, some of which we have covered above and some that are specific to grid and flexbox. Note as you don't difine columns or rows in a template in flexbox, you would not use them in the same way as grid.

In the examples below we are using various units in grid-template-columns to define a three column grid. A fixed width track (a row in column-based template or column in a row-based template) can be difined with any of the units from above. One option for a more dynamic track is a percentage, but unless there is a good reason to use it, percentage should be avoided due to content overflow.

Let's begin with a grid specific unit!

Grid fraction unit

fr stands for fraction, each element with a fraction unit will gain a proportion of the remaining space after fixed elements are calculated. This space will be based on the factor before the fr. i.e. if there is 60px remaining space and there are 3 elements each with 1fr, each one will get 20px (60px / 3 i.e. 1fr * 3 elements), if the first has 2fr it will get 30px (60px / 4 * 2 i.e. now there are 4 fractions in total and this element has to of them ) and the other two will get 15px.

If not used in a function (see below) then it implies an automatic minimum (see auto below).

Lession 4 - Resizing

Reactions to size changes

So what? Who cares about all this stuff? In the past we trended to make grid that were even or 2/1 ratio columns (i.e. 1 column that is twice as big as the others). What we can start to do with Modern css is to allow the layout to be responsive to the intrinsic size of the content. Responding in a predictable way to a piece of unknown content loaded from a CMS for example. As a developer we don't know ahead of time what the content writer is going to add, so we need to make a layout that can handle a lot of text or a little gracefully.

We can also create layouts that response to content in different ways that maybe more interesting than everything growing or shrinking at the same rate.

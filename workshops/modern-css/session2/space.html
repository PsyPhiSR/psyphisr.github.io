Lession 1 - Units

Alternatives to px

As developer's we have a tendency to copy a pixel value from a design and use that in the site, only to find that it does not work when we switch to a different screen size. This lession explores what options we have as developers to use different css units types.

Below we have a box with options available on a width parameter. In the past, percentages were used to create even columns (50% for 2, 33.3% for 3, etc.) we will see that now we have better options for this.

Note that rem and em are the same size, both are based on 16px base font size. ex is the x-height of the current font, which is defined as the height of a lowercase 'x' (fonts of the same font size can have different x-heights. see below) ch is width of the "0" (zero).

How font family affects sizes

Below shows an x in Barlow and Inter with visual guidelines at the top and bottom of the Barlow x. Note how the Inter x is a little taller for both lowercase and uppercase. Also the two fonts have a different baseline, to align them I added align-items: baseline to the parent. Try removing it to see what happens.

The boxes above show that rem and em are not affected by the font-family but ex and ch are. As ex is related to the height it would be more logical to use it to affect the height than the width, but there may still be times when it could be used.

How font size affects sizes

The grids below shows what units are affected by font size, in the second grid the font size has been doubled. Note that this change does not affect the rem unit. rem stands for "root" em unit so is the font size set on the html, so if you add font size to the html, this size will change.

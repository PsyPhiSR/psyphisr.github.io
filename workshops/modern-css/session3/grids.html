Introduction

In this session we explore the fundamental question...

Should I use Grid or Flexbox?

we will also explore the question...

Do we need both?

We will look at the main differences between the two, and what might influence which one you choose (hint we do need both). Let's start with a quick recap on the two techniques.

All of the red boxes below have a few additional props.
min-height: 5vmin;
min-width: 5vmin;
The container div also has padding: 2rem applied to it.

Most of the examples below are a container div with three child divs, unless overwise stated. I am using red outlines to show the outlines of the divs, ignore the thicker lines it's just where two outlines are next to each other and has no significances.

This session will only look at even columns examples, see the details on the Space session to make columns that have difference proportions.

Grid

To use grid all you need is a div with display: grid;

Note that the default is for the divs to stack in a column and are the full width of the container.

To dynamically make multiple columns we can add the following to the container.
grid-auto-flow: column;

Note by default each div will fill an equal fraction of the space available. Just as if the with had been fixed at 1fr.

We can also use grid-template-columns: 1fr 1fr 1fr; to set the number of columns. The key word here is set as it stops being dynamic.

Note with three child divs this looks exactly the same as the grid-auto-flow: column; example above.
Note that if we repeat the two examples for auto and template but with only two child divs we can see that the grid-auto-flow: column; example expands to fill the container but template leaves a space at the end.

Flexbox

To use flexbox all you need is a div with display: flex;

Note that the default is for the divs to line up in a single row and shrink to fit their contents.

We can add flex-grow: 1; to each child to make them expand to the full width and it now looks the same as grid-auto-flow: column; above. It behaves the same way when there are only two columns. However, there is not an easy way to shrink it to make a space of equal proportion as we did in the grid template example. Yes you use percentages, but see the Space session to find why this might be a bad idea.

We can add flex-direction: column; to the container to switch to a column.

Note that each of the divs now expands to the full width of the container.

Differences In One Dimension

So far, we have only looked at a container with either a single row or column of children, but we can already seen that there are differences between grid and flexbox.

The main differences are the default direction, how the children are sized and where the layout definition is placed.

* Note when in columns both grid and flexbox child width defaults to full width of the container.
** You can add flex-grow: 1; use a fraction of the space the same as grids 1fr.

Two Dimensions

Lets look at the two techniques when we have more than one row.

Most of the examples below are a container div with five child devs, with the prefered arrangement of three even columns.

Multi-Row Grids

To make the grid multiple rows we can use grid-template-columns as we did above.

Multi-Row Flexbox

This is where we see a big difference between grid and flexbox. Flexbox is inheritly one dimensional, this does not mean that it will not go onto multiple rows, just that you have less control.

We must add flex-wrap: wrap; to make the flexbox flow over to multiple rows. We must also fix the width of the children, otherwise, flexbox will try to fit them into a single row.

Ah!! I used percentages. It's a hack, but it is not quite the same as the grid-template-columns example above. However, 50% or 25% maybe useful sometimes.

Or you can give the children a fixed width.

Initial Conclusion

So all this might suggest that it doesn't really matter which one you choose if you have a single row (it doesn't matter!). Or that grid is just a better tool as it is more predictable.

However, there are circumstances where one option is better than the other. So let us look at them.

Grid - The King of Page Layout

We can use this css to create a centered column for content with an even border either side. This setup does not require a media query to work on mobile, when the screen is small enough the minimum --padding kicks in to leave a nice centered column in the middle.
grid-template-columns:
 minmax(var(--padding), 1fr) //gutter
 min(var(--max-content-width), (100% - (2 * var(--padding))))
 minmax(var(--padding), 1fr); //gutter
> * {
 grid-column: 2;
}

Flexbox Examples

One problem with flexbox is that most of the definition needs to go on the children, to combat this we can write rules like this (this is the css for the third example below).

.flex-pancake>* {
 flex: 1 1 var(--flex-coumn-width);
}

For this first example each div has a fixed width. If the container can accommodate all of the divs, they will go on one row. Otherwise, the extra div will drop to the next row.

Note the behaviour is that same as the Grid example with a fixed number of columns. But in the grid example the divs expand to the fill the whole width of the container, with the dropped div matching the width of the other divs.

In the second example, each div has been given a minimum width, if the container is wide enough to contain all of the divs with at least their minimum width, then there are displayed in a single row. As soon as the container is too small, the last div is dropped to the next row and grow to fill the available space.

There is a design decision here, with flexbox the dropped div is bigger than the rest, and may be perceived as more important, this may not be what you want.
